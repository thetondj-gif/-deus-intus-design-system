# Deus Intus Design System

**Awaken the Power Within.**

This repository is the public design-system package for Deus Intus. It gives agents, designers, and future tools a reusable brand source that stays aligned with the canonical Deus Intus operating system.

## Package purpose

The package defines the Deus Intus visual system for:

- sacred-modern interfaces
- storefront and collection layouts
- product storytelling surfaces
- social and editorial brand assets
- reusable HTML previews and UI kits

It is designed to feel premium, cinematic, spiritually intelligent, and commercially usable.

## Core files

| File or folder | Purpose |
| :--- | :--- |
| `open-design-system.md` | High-level public brand system for external tools. |
| `DESIGN.md` | Source-backed design rules and implementation guidance. |
| `SKILL.md` | Agent-facing usage instructions for applying the system correctly. |
| `colors_and_type.css` | Shared palette, typography, spacing, motion, and component tokens. |
| `preview/` | Auditable previews for colour, typography, spacing, layout, components, product, and brand moments. |
| `ui_kits/app/` | Applied interface kit built on the shared tokens and components. |
| `context/github/` | Evidence notes for GitHub-linked source context. |

## Brand summary

The current system is anchored in:

- Deep Navy and Midnight luxury surfaces
- Sacred Gold hierarchy used with restraint
- Aqua energy accents for signal and lift
- Bone text and calm editorial contrast
- Italiana, Lora, Montserrat, and Jura typography
- quiet sacred geometry and breath-like motion

This package should feel like a premium ritual tool, not a default SaaS kit and not generic spiritual content.

## How to use with external tools

Use the raw GitHub URL when a tool needs the core public design brief:

```text
https://raw.githubusercontent.com/thetondj-gif/-deus-intus-design-system/main/open-design-system.md
```

For tools that can inspect a full repository, point them to:

```text
https://github.com/thetondj-gif/-deus-intus-design-system
```

## Usage rules

1. Keep the tagline exact: **Awaken the Power Within.**
2. Use the tokens in `colors_and_type.css` instead of inventing local palette variants.
3. Preserve generous spacing and clear hierarchy.
4. Use sacred geometry subtly and mathematically cleanly.
5. Keep copy grounded, empowering, and free of hype.
6. Make previews and UI kit examples feel calm, premium, and immediately recognisable as Deus Intus.

## Audit note

This repo intentionally includes docs, tokens, previews, and an applied UI kit so external agents can use the brand without access to the main private application codebase.
